**Date:** February 12, 2026

## Competitors Analyzed

## 1. FREED

### Pricing
| Plan | Price | Key Features |
|------|-------|-------------|
| Starter | $39/mo | 40 notes/mo, specialty templates, live support |
| Core | $79/mo | Unlimited notes, live onboarding, AI assistant (beta) |
| Premier | $104-119/mo | EHR push, prior visit summaries, AI chat, patient instructions, ICD-10 coding |
| Groups | Custom | Shared patients (alpha), team templates, SSO, dedicated manager |

### Strengths ✅
- **"Learn my format"** — Edit one note, Freed learns your style forever. One-click template creation from existing notes
- **EHR Push** — Direct integration to push notes into EHR systems (Premier tier)
- **Prior visit summaries** — AI pulls context from previous encounters automatically
- **AI Chat agent** — Ask questions about previous notes/patient history
- **Magic Edit** — Natural language editing ("make the assessment more detailed")
- **ICD-10 coding** — Auto-suggests codes from encounter context
- **Patient instructions** — Auto-generates plain-language follow-up emails
- **Post-visit letters** — 14+ letter types (referrals, absence notes, return-to-work)
- **Zoom app integration** — Joins Zoom calls directly
- **17 specialty-tuned learned templates** in their library (Allergy/Immunology, Endocrinology, Gastro, Geriatrics, Neurology, Podiatry, Pulmonology, Rheumatology, Emergency Med, Medicine, Mental Health, OB/GYN, Pediatrics, Psychiatry, Psychotherapy, Surgery, Default)
- **Structured + Learned template types** — Two template paradigms
- **50% student discount**
- **7-day free trial**, no credit card

### Weaknesses ❌
- **No rehab/PT/OT/SLP templates** — Zero rehabilitation-specific coverage
- **Limited template library** — Only 17 specialty templates, all "learned" (black box)
- **No procedure note framework** — Missing H&P and procedure documentation
- **Pricing is high** — $119/mo for full features, $79/mo for just unlimited notes
- **No AI Receptionist** — Single-product focus (scribe only)
- **No fax/insurance automation** — Narrow feature scope
- **Template customization is basic** — Sections and prompts, but no granular field-level control
- **No Chrome Extension for recording** — Web app only + mobile
- **Shared patients still in alpha** — Team features immature
- **Audio temporarily stored** — Not instant deletion (privacy concern for some)

## OMNISCRIBE — Current State & Comparison

### What OmniScribe Has That Others Don't ✅
- **Evidence-based documentation frameworks** — Not just templates, but regulatory-grounded, item-level documentation with sourced clinical guidelines (AMA CPT, CMS, Joint Commission, etc.)
- **Three framework types** — SOAP + H&P + Procedure Note (no competitor has all three)
- **Specialty-specific add-ons** — 8+ specialty modules (Cardiology, Pulmonology, GI, Neuro, Rheumatology, Derm, Endo, PM&R) with evidence-based items
- **Provider-type awareness** — MD/DO/PA-C/NP/DNP with aliasing logic
- **Subtype depth** — 5 subtypes per framework (15 total), 30-50 items each
- **Medical necessity built-in** — Every framework includes medical necessity documentation
- **Rehab frameworks (building)** — PT/OT/SLP coverage = massive gap none of them fill
- **Behavioral health frameworks (building)** — Deeper than Twofold's template approach

### What OmniScribe Needs to Compete ❌
- **Live recording/transcription** — All three competitors capture audio in real-time
- **"Learn my format" capability** — Freed's killer feature
- **EHR integration** — At minimum copy-friendly, ideally direct push
- **ICD-10/CPT coding suggestions** — Freed has this, others don't yet
- **Patient instructions generation** — Freed has this
- **AI chat/prior visit context** — Freed's Premier feature
- **Template marketplace/library** — Twofold's 60+ templates show this matters
- **Web/mobile app** — Currently HTML documentation, needs a product UI
- **Pricing strategy** — Market is $39-119/mo per clinician

## Key Strategic Insights

### 1. REHAB IS A WIDE OPEN MARKET 🎯
**None of the three competitors have any rehabilitation-specific templates.** No PT eval, no OT note, no SLP documentation, no FIM scoring, no 8-minute rule compliance, no APTA-aligned frameworks. OmniScribe's rehab frameworks will be a genuine first-mover advantage. Rehab clinics are underserved and desperate for AI scribing.

### 2. BEHAVIORAL HEALTH IS CONTESTED BUT SHALLOW
Twofold has the most BH templates (20+), but they're all section-header templates (SOAP, BIRP, DAP, etc.) — no evidence-based item-level depth. OmniScribe's BH frameworks with DSM-5-TR integration, validated instruments (PHQ-9, GAD-7, C-SSRS), 42 CFR Part 2 compliance, and ASAM criteria will be significantly deeper.

### 3. THE FRAMEWORK APPROACH IS THE MOAT
Competitors sell templates (section headers with AI fill). OmniScribe sells frameworks (evidence-based, regulatory-sourced, item-level documentation). This is a fundamentally different value proposition — compliance-grade documentation vs. "good enough" notes.

### 4. PRICING SWEET SPOT
- Entry: $39/mo (match HealOS)
- Standard: $79/mo (unlimited, match Freed Core)
- Premium: $99/mo (undercut Freed Premier with more features)

### 5. MUST-BUILD FEATURES (Priority Order)
1. **Audio capture + transcription** (table stakes)
2. **Web app with recording UI** (can't just be HTML docs)
3. **Template selection + note generation** from frameworks
4. **"Learn my style" adaptation** (Freed's differentiator)
5. **EHR push/integration**
6. **ICD-10/CPT coding from encounter**
7. **Patient instructions + letters**
8. **Prior visit context + AI chat**

## DETAILED UX/UI ANALYSIS

### FREED — UX Deep Dive

**Visual Design:** Clean, minimal, professional. Muted purple/white color scheme. Sidebar navigation is icon-based (Visits, Templates, EHR Push, Subscribe) — compact but requires learning what each icon means. Overall feels like a polished SaaS product.

**Recording Flow (3 steps to start):**
1. Top bar: Select patient name (combobox) + set pronouns + choose "New Patient" or "Returning Patient" template
2. Click purple "Capture conversation" button (top right, always visible)
3. That's it — recording starts

**Ease Rating: 9/10** — Dead simple. Patient selector is right there, template picker is a dropdown, one big button to start. The "Returning Patient" vs "New Patient" toggle at the top is smart UX — most visits are one or the other.

**Note Output Layout:**
- **Visit Summary** at top — one paragraph overview (great for quick review)
- **Subjective** — Full narrative with Medical History, Medications, Social History, ROS pulled out as sub-sections within the section
- **Objective** — Vital signs + physical exam
- **Assessment & Plan** — Combined with differential diagnosis and action items
- **Patient Instructions** — Auto-generated plain language (marked "Premier")
- Each section has: 👍/👎 feedback buttons + Copy button + collapsible toggle
- Sections are editable inline (click to type)
- Section headers are editable textboxes (you can rename them!)
- "Ask anything" chatbot docked at bottom — voice or text input for editing

**Killer UX Features:**
- **Tabs: Context | Note | Transcript | + New letter** — Four views of the same encounter, tab-based
- **AI Chat sidebar** — "Chat about John Doe" — ask questions about the note, request edits
- **Copy all** button (top right) — one click to clipboard entire note
- **Resume** button — can re-open recording if you forgot something
- **Template dropdown right in the note view** — switch templates without leaving
- **Pronouns selector** — He/him, She/her, They/them — progressive, and affects note language
- **Saved status indicator** — "Saved seconds ago" always visible

**Weaknesses in UX:**
- Left sidebar is icon-only (no labels on desktop until you hover) — slightly confusing on first use
- No visible "end recording" state shown — have to remember to stop
- Patient Instructions section is locked behind Premier (shows purple "Premier" badge) — feels paywalled
- No batch operations on notes list
- AI chat takes up note real estate (but is collapsible)

### TWOFOLD — UX Deep Dive

**Visual Design:** Bright blue/white scheme. More playful/friendly than Freed. Left sidebar is a thin icon rail + a notes list panel. Clean card-based template library. Intercom chat widget always visible (bottom right) — they're investing in support.

**Recording Flow (3 steps to start):**
1. Patient Name text field (center of screen)
2. Template dropdown (defaults to "SOAP")
3. Modality dropdown ("In person" / "Telehealth")
4. Click blue "Capture Conversation" button
Also: "New to Twofold? Try a demo session" link for onboarding

**Ease Rating: 8/10** — Very clean, but one more dropdown than Freed (modality). The centered layout makes the recording screen feel calm and focused. "Try a demo session" is smart onboarding.

**Note Output Layout:**
- Patient name + "Tx" button (transcript access) at top
- "Magic Edit" + "Copy All" + "..." menu buttons (top right)
- Each section (Subjective, Objective, Assessment, Plan) in its own **card with textarea**
- "Copy Text" button per section
- "Save Changes" button under each textarea
- Notes are fully editable in-place (textarea, not contenteditable)
- Left panel shows all notes chronologically with patient name, title, duration, date

**Killer UX Features:**
- **60+ templates** — Massive library, searchable, organized with "My Templates" + "All Templates" tabs
- **Edit + Share buttons on every template** — Social/team feature built into templates
- **Template preview showing sections** — See exactly what sections a template has before selecting it
- **Platform-specific templates** — Simple Practice, Headway, Grow Therapy, Rula (therapists already use these EHRs, so ready-made templates match their workflow)
- **Note Settings page** — Spoken language, Note language, patient reference style (by name vs generic), clinician reference, timezone, include previous notes toggle
- **"Include previous patient notes in context"** — toggle in settings (not paywalled!)
- **Specialty dropdown in settings** — Physical Therapy was an option (they support rehab in settings but NOT in templates!)
- **Patients page** — Separate patient management section

**Weaknesses in UX:**
- **Textareas for note sections** — feels older/clunkier than Freed's inline editing. You're editing in a box, not flowing text
- **No section-level feedback** (no 👍/👎 like Freed)
- **No AI chat** — can't ask questions or conversationally edit
- **No visit summary** — jumps straight into SOAP sections
- **No transcript tab** — transcript access is via small "Tx" button
- **No patient instructions** — nothing generated for patients
- **Copy is per-section only** (plus Copy All) — no smart formatting
- **Notes list is basic** — no status indicators, no filters beyond search + date
- **No "Resume" recording** option visible
- **Settings are minimal** — only 6 options total

### HEALOS — UX Deep Dive

**Visual Design:** Purple/dark purple sidebar, white content area. More "enterprise" feeling than the others. Has an "Old UI" toggle (meaning they recently redesigned). Sidebar is labeled (not icon-only) — easier to navigate. "NEW" and "BETA" badges show active development.

**Recording Flow (2 steps to start):**
1. Search/add patient name (center search bar)
2. Select patient → recording starts
Right sidebar shows "Recent Sessions" and "YOUR SESSIONS" with "+ New Visit"

**Ease Rating: 7/10** — Requires patient selection before recording (can't record without a patient). This is more structured than Freed/Twofold but adds friction. The "Select or add patient to start recording" prompt with illustration is clear though. Good that it shows previous visit context.

**Template Library (26 predefined, 5 pages):**
Page 1: Elation EHR, Charm Health, Follow-Up Patient Visit, New Patient Visit, ECW Template, Athena Template
Page 2: Utilization Review, Intake Template, Simple SOAP, Focused Emergency Dept Visit, Return Addiction Med Visit, New Addiction Med Visit

**Killer UX Features:**
- **"Generate Template" button** — AI-powered template creation (unique feature, no competitor has this!)
- **EHR-specific templates** — Elation, Charm Health, ECW, Athena built-in (maps to actual EHR section structures)
- **Dot Phrases** — EHR-style shorthand (Configure > Dot Phrases) — power user feature
- **Keywords** — Custom keyword triggers (Configure > Keywords)
- **Appointments page (NEW)** — Calendar/scheduling integration
- **Tutorials page** — Built-in learning (onboarding)
- **AI Receptionist (BETA)** — Phone answering right in the same app
- **Chrome Extension** — Record from any browser tab
- **Organization support** — "No organization selected" dropdown = multi-clinic ready
- **Session counter** — "20/20 Sessions" shown prominently (clear usage tracking)
- **Template Preview + Save** — Can preview before saving to your library
- **Predefined + Custom tabs** — Clear separation
- **"Notes" filter dropdown** — Can filter template types

**Weaknesses in UX:**
- **Clunkiest recording flow** — Must add patient first, no quick-start option
- **No sample/demo note** — New users see empty state immediately (Freed shows example, Twofold has sample)
- **404 errors on direct navigation** — URL routing seems fragile (I hit multiple 404s)
- **Template sections are text-only descriptions** — No visual preview of what the note will look like
- **No "learn my format"** — Manual template creation only
- **Old UI toggle visible** — Signals instability/transition
- **"5 Days Left" badge** — Aggressive trial pressure in the header
- **No note editing UI visible** — Couldn't see how generated notes look (no sample data)
- **Sidebar feels cluttered** — AI Scribe + AI Receptionist + Configure + Chrome Extension + Trash + Account Settings all crammed in

### UX RECOMMENDATIONS FOR OMNISCRIBE

Based on the deep dive, here's what OmniScribe should steal/improve:

**From Freed (best overall UX):**
1. Visit Summary at top of every note — great for quick scan
2. Inline editing (contenteditable, not textareas)
3. 👍/👎 feedback per section — teaches the AI
4. AI chat sidebar for conversational editing
5. Patient Instructions auto-generation
6. "Resume" recording capability
7. Pronouns selector
8. Tab-based views (Context | Note | Transcript | Letters)

**From Twofold (best template ecosystem):**
1. Massive pre-built template library organized by specialty AND platform (Simple Practice, Headway, etc.)
2. Template sharing between team members
3. "Include previous patient notes in context" as a simple toggle
4. Template section preview before selecting
5. Platform-specific templates matching EHR workflows
6. Note Settings page with language, reference style, timezone

**From HealOS (most innovative features):**
1. "Generate Template" — AI creates templates from description (nobody else has this)
2. Dot Phrases — EHR-style shorthand expansion
3. EHR-specific templates (Elation, Athena, ECW)
4. Chrome Extension for recording
5. Organization/multi-clinic support
6. Appointments integration

**OmniScribe's unique advantages to emphasize in UI:**
1. **Framework selector** (not just template) — Show users they're getting evidence-based, regulatory-grounded documentation, not generic SOAP
2. **Medical necessity built into every note** — Highlight this, no competitor does it
3. **Subtype picker** — After selecting framework, pick the specific subtype (New Patient vs Follow-Up vs AWV etc.)
4. **Source citations** — Show the regulatory source for each item (builds trust, supports audits)
5. **Provider-type awareness** — UI adapts based on MD/DO/PT/OT/SLP/LCSW credential
6. **Rehab + BH as first-class citizens** — Not afterthoughts like competitors
